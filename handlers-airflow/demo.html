// Stage by Stage

How Data Moves Through the System

Six sequential stages take raw YouTube data from API response to query-ready analytics, all running automatically.

01
YouTube API + Whisper Extraction
Airflow DAG connects to the YouTube Data API and collects video metadata — titles, descriptions, view counts, likes, comments, duration, and publish dates. OpenAI Whisper transcribes each video's audio track. Metadata lands in S3 as JSON; transcriptions as Parquet.
Apache AirflowYouTube APIOpenAI WhisperAWS S3
Complete
02
Event-Driven Trigger — Lambda + Kinesis
An AWS Lambda function monitors S3 for new file arrivals. The moment data lands, it publishes an event to Amazon Kinesis which triggers the Databricks processing cluster. No polling, no wasted compute — processing fires only when data is actually present.
AWS LambdaAmazon KinesisAPI GatewayAWS Glue
Complete
03
Bronze Layer — Raw Ingestion
Databricks writes all incoming data to the Bronze layer exactly as received — no schema enforcement, no transformations. This immutable layer preserves the full history and allows any downstream layer to be reprocessed from source at any point.
DatabricksPySparkAWS S3Unity Catalog
Complete
04
Silver Layer — Structure and Trust
Spark jobs unpack nested JSON payloads, resolve duplicate records, standardize data types, and run automated quality validation. Only records that pass all checks advance. Failed records are flagged and quarantined for review without affecting the Bronze source.
PySparkSpark SQLDatabricksGlue Catalog
Complete
05
Gold Layer — dbt Dimensional Modeling
dbt Cloud builds the analytics-ready Gold layer: a Snowflake schema normalized to 3NF with dimensional models for videos, channels, and engagement metrics. Source freshness checks, referential integrity tests, and null constraints run on every deployment. All logic is version-controlled SQL.
dbt CloudSnowflake Schema3NFAWS Athena
Complete
06
Infrastructure as Code
All infrastructure — S3 buckets, Lambda functions, Databricks clusters, VPCs — defined in Terraform and provisioned reproducibly. Ansible manages configuration. Docker handles local environments. GitHub Actions runs CI/CD on every commit.
TerraformAnsibleDockerGitHub Actions

// Medallion Architecture

Three Layers, One Source of Truth

The Medallion pattern separates ingestion, validation, and modeling into discrete layers. Each layer has a clear contract with the layers above and below it.

🥉
Bronze
RAW · IMMUTABLE · APPEND-ONLY
Every record written exactly as received from source. No schema enforcement, no transformations. This is the permanent, replayable record of everything ever ingested.
JSON
Metadata format
Parquet
Transcript format
🥈
Silver
STRUCTURED · VALIDATED · DEDUPED
Spark jobs enforce schema, resolve duplicates, standardize types, and run automated quality checks. Only validated records advance. Failed records are quarantined without touching Bronze.
100%
Test coverage
3NF
Normal form
🥇
Gold
ANALYTICS-READY · DBT-TESTED
dbt Cloud builds dimensional models and computes engagement metrics. Referential integrity tests run on every deployment. Analysts query this layer directly via Athena.
